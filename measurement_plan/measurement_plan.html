Events implemented

  • 01
    page_view
    Fires on every page load (DOMContentLoaded). Reads window.TNK_PAGE defined per page. Sends page name, category, type, content group, and author.
  • 02
    select_content
    Fires on all navigational clicks — nav links, hero CTAs, project/blog cards, filter pills, social links. Uses delegated click listener on a[data-track] and tnkNavigate() helper.
  • 03
    generate_lead
    Fires on contact form submit. Dynamically collects non-PII fields. Skips password/tel/hidden and PII name patterns. Textarea → word_count + char_count. Email hashed via SHA-256 (crypto.subtle).
  • 04
    orbit_interaction
    Fires on hover of the orbiting skills section on the Home page. Sends orbit_pause (boolean, animation state) when entering/leaving the stage, and skill_hover (tooltip text) per icon hover.
  • 05
    search
    Fires 500 ms after user stops typing in the Blog search bar (min 2 chars). Sends search term, results count, has-results flag, term length, section, and active filter pill.
  • 06
    post_engagement
    Fires on blog post pages when a visitor likes, unlikes, or submits a comment. Sends content_id (post slug), content_name (post title), and action ("like" | "unlike" | "comment"). Like state is persisted per browser via localStorage UUID + Supabase.

Versioning rules

A new version is created for every major update: new event added, existing event removed or renamed, new parameter added/removed, tracking approach changed. Minor fixes (typos, broken links, screenshot refreshes) do not increment the version. Before each version bump, the current file is archived to archive/measurement_plan_vN.html.
